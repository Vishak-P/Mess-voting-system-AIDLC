# User Personas
## Mess / Canteen Menu Voting System

---

## Persona 1: The Student — "Priya"

| Attribute | Detail |
|---|---|
| **Name** | Priya Patel |
| **Role** | Student (default role) |
| **Age** | 19 |
| **Context** | Lives in college hostel, eats all three meals at the canteen |
| **Tech Comfort** | High — uses smartphone daily, comfortable with web apps |
| **Primary Goal** | Vote for the food she wants and see what's being served |
| **Pain Points** | Doesn't know what's for lunch until she gets there; feels her food preferences are ignored |
| **Motivations** | Wants her voice heard; curious to see if her preferred dish wins |

### Key Workflows
1. Log in → view today's menu → vote for preferred dish per meal
2. Check results after deadline to see if her dish won
3. Submit feedback/rating after eating a meal
4. Browse next week's menu to plan ahead

## Persona 2: The Mess Admin — "Mr. Sharma"

| Attribute | Detail |
|---|---|
| **Name** | Mr. Sharma |
| **Role** | Admin |
| **Age** | 45 |
| **Context** | Manages the college canteen; responsible for weekly menu planning |
| **Tech Comfort** | Moderate — comfortable with web dashboards, not a developer |
| **Primary Goal** | Understand student preferences and plan menus accordingly |
| **Pain Points** | No visibility into what students actually want; manual feedback collection is tedious |
| **Motivations** | Reduce food waste; improve student satisfaction; make data-driven menu decisions |

### Key Workflows
1. Log in → create weekly menu with dish options and voting deadlines
2. Monitor live voting activity on the dashboard
3. Lock voting after deadline or manually when needed
4. Export results as CSV for weekly planning meetings
5. Promote a student to admin role when needed
6. Copy last week's menu as a template for the new week
